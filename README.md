**Nautilus** 是一個專為「物理隔離」與「邊緣運算」設計的高性能路由引擎。它的核心設計目標是透過 **Unix Domain Sockets (UDS)** 取代傳統的 TCP 網路棧，為現代微服務提供一個不可掃描、零延遲且具備強大調度能力的底層通訊架構。

在傳統架構中，服務之間透過 TCP 端口溝通。Nautilus 則將所有通訊鎖在檔案系統的權限控制之下。

---

## 核心設計哲學

### 1. 全 UDS 化 (Full-UDS Architecture)
Nautilus 核心完全不監聽任何 TCP 埠位。所有的入口與出口皆透過 `.sock` 檔案進行。這意味著：
* **物理級隔離**：除非擁有特定目錄的存取權限，否則外部無法掃描或嗅探服務。
* **低開銷**：繞過了 TCP 的封包與校驗機制。

### 2. 控制面與數據面分離
Nautilus 採用了類似軟體定義網路（SDN）的架構，將複雜的邏輯與純粹的轉發徹底拆分：
* **Nautilus UI (控制面)**：負責解析 YAML 配置、編譯路由邏輯，並產出二進位快照。
* **Nautilus Core (數據面)**：極度輕量化的執行緒，只負責根據快照進行流量轉發，不處理 UI 或複雜的配置解析。

## 技術亮點

### 編譯型路由與原子更新
不同於傳統代理在每次請求時進行複雜的動態匹配，Nautilus 引入了「路由編譯」概念：
1.  **YAML 編譯**：將易讀的路由規則編譯成高效的 **Radix Tree**。
2.  **GOB 序列化**：將樹狀結構序列化為二進位 `GOB` 檔案。
3.  **Atomic Swap (原子切換)**：核心偵測到新檔案時，利用記憶體指針進行瞬時切換，確保更新配置時「零停機」且請求不中斷。

### 強化安全性的內部端點
Nautilus Core 具備一個隱藏的 `internal.sock` 控制端點。管理端 UI 透過這個端點主動下發重載指令或監控流量狀態。這條線路不經過網路介面，完全依賴 Linux 權限體系加固。

---

## 生態系組件

* **Nautilus Core (Go)**：無狀態的高性能轉發內核。
* **Nautilus UI (Go)**：具備規則編輯、視覺化路由樹與編譯功能的管理中心。
* **Relay Sidecar (Rust)**：高性能適配器，負責將 UDS 流量轉譯給傳統的 TCP HTTP Server。
* **Entrypoint (Rust)**：選配組件，負責將公網 TCP 流量安全導入 UDS 體系。

## Nautilus 開發路線圖

### 第一階段：核心模型與 UDS 基礎建設
本階段目標在於建立最基礎的 Unix Domain Socket 轉發能力，並驗證雙端點通訊的可行性。

* **定義核心資料結構**：建立基礎的路由節點與服務端點結構，初步定義 UDS 路徑與目標服務的映射關係。
* **實現基礎 UDS 代理**：開發核心轉發邏輯，利用 `io.Copy` 在兩個 UnixConn 之間進行位元組流傳遞。
* **環境模擬工具**：編寫簡單的測試腳本，模擬後端 HTTP Server 監聽 UDS 並回傳響應，驗證數據鏈路完整性。

---

### 第二階段：路由編譯器與 GOB 序列化
本階段引進編譯概念，將易讀的配置轉換為機器高效處理的二進位格式。

* **YAML 配置定義**：設計包含 Host、Path 與布林邏輯規則的 YAML 格式。
* **布林表達式解析器**：整合或實作邏輯運算解析，將字串規則編譯為記憶體中的判斷樹。
* **GOB 序列化模組**：實作將記憶體路由樹寫入磁碟 GOB 檔案的邏輯，確保類型預註冊以提升效能。
* **原子替換機制**：在 Core 中實現 `atomic.Pointer` 邏輯，當偵測到 GOB 檔案更新時，進行無鎖的路由樹切換。

---

### 第三階段：控制面與數據面分離
將系統拆分為獨立的組件，並建立內部控制通道。

* **組件拆分**：將代碼庫建構為 `nautilus-ui` 與 `nautilus-core` 兩個獨立的執行檔。
* **內部控制端點**：在 Core 中啟動 `/services/nautilus-internal.sock`，專門接收來自 UI 的管理指令。
* **指令集實作**：定義一套輕量級協議，讓 UI 能透過該端點下發重載指令、查詢當前路由樹版本及流量統計。
* **雙重監聽機制**：UI 監聽 YAML 變動並觸發編譯；Core 監聽內部端點指令並執行加載。

---

### 第四階段：Rust 適配器與邊緣入口
利用 Rust 的高性能特性處理網路邊界與傳統協議的對接。

* **Rust Relay-Sidecar**：開發獨立的 Rust 服務，負責 UDS 到 TCP 的轉譯，讓傳統 HTTP Server 無縫接入。
* **Rust TCP Entrypoint**：開發選配的入口組件，負責將外部 TCP 流量安全地導入 Nautilus 的 UDS 入口。
* **性能基準測試**：針對「全 UDS 鏈路」與「含 Rust 適配器鏈路」進行延遲與吞吐量分析。

---

### 第五階段：Web 管理介面與可視化
提升運維體驗，將複雜的邏輯可視化。

* **路由樹可視化**：在 UI 介面展示當前編譯後的 Radix Tree 結構。
* **布林規則編輯器**：提供具備語法檢查的規則編輯區域，支援 Traefik 風格的函數運算。
* **部署模擬器**：實作「預覽編譯」功能，在正式更新 GOB 前，模擬特定請求路徑的匹配結果。
